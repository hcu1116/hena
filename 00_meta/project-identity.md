## 0. 이 문서는 뭐예요?

학교에서 집까지 오는 동안 논의한 모든 내용을 한곳에 모은 문서입니다.
- 프로젝트가 흔들릴 때 **돌아올 기준점**
- 나중에 **GitHub README의 베이스**
- 면접 준비할 때 **꺼내 보는 자료**

> ⚠️ 이 문서는 v0.1 초안입니다. 빈 칸이나 "본인이 채울 부분"은 직접 본인 언어로 채워야 진짜 자산이 됩니다.

## 1. 프로젝트 한 줄 정의

> 기면증 환자의 일상 안전을 위한 웨어러블 기반 실시간 졸음·탈력발작 위험 예측 시스템 — 일반 사용자에게는 졸음 관리 도구로도 작동

**본인 말로 다시 써보기:**
```
기면증 같은 희귀병을 앓고 있는 환자들이 일상속에서 갑자기 잠들거나 쓰러질 위험에 대비하기 위해서 웨어러블 데이터로 위험 시점을 예측할 수 있고 예방할 수 있게 미리 알려주는 시스템
```

---

## 2. 왜 이 프로젝트인가 (동기)

- **개인적 동기:** 가까운 지인이 기면증으로 힘들어하는 모습을 보며, 기술로 일상의 위험을 줄일 수 있지 않을까 생각함
- **면접용 버전:** "주변 지인 중 기면증으로 힘들어하시는 분이 계셨고, 일상에서 작은 사고들을 보면서 '기술로 일상의 위험을 줄일 수 있지 않을까' 생각했습니다."
- **커리어 동기:** AI + 헬스케어 분야 포트폴리오 구축, 장기적으로 운동 AI(보디빌딩 심사·운동 보조기구)로 확장

## 3. 포지셔닝 (확정)

**A 메인 + B 보세 하이브리드**

| 구분 | 내용 | 역할 |
|---|---|---|
| 메인 (A) | 기면증 환자의 일상 안전 보조 | 면접 첫 문장, 핵심 메시지 |
| 보세 (B) | EDS(과도한 주간졸림) 일반 사용자로 확장 가능 | 시장성 질문 방어용 |

**시장 근거:**
- 한국 기면증 환자: 약 4천~2만명 (희귀질환)
- 한국 기면증 치료제 시장: 2023년 8,670만 달러 → 2030년 1.8억 달러 (연 11% 성장)
- EDS 일반으로 넓히면 타겟 1,000만 명+ (수면부족·교대근무·만성피로 포함)

**현황 (선행 연구 조사 결과):**
- 기면증 진단 AI는 대부분 PSG(병원 수면검사) 기반 → 웨어러블 기반은 아직 초기 단계
- 상용 제품(갤럭시·애플·핏빗·오우라) 중 기면증 특화 기능은 없음
- → **"일상 보조" 포지셔닝은 거의 비어있는 영역**

---

## 4. 페르소나

### 메인 페르소나: 진단받은 기면증 환자
가장 힘들어하는 것 (실제 사례 기반):
1. 약물 부작용 — 복용 후 기억 소실, 기분 급변동, 식욕 폭증
2. 탈력발작(Cataplexy) — 샤워·양치·설거지 중 갑자기 쓰러짐 → 부상
3. 극심한 주간졸림 — 일상 전반의 사고 위험

### 서브 페르소나: EDS로 일상이 힘든 일반 사용자
- 수면부족, 교대근무자, 만성피로 등

---

## 5. 핵심 기능 (우선순위)

1. **졸음 위험도 정량화** — HR/HRV/SpO₂/활동량 → 위험도 점수 [0,1]
2. **탈력발작 위험 시간대 예측** ⭐ (차별화 핵심 — 아무도 안 함)
3. **미세수면(Microsleep) 감지 및 경고**
4. **약 복용 효과 추적** — 의사 상담용 객관 데이터
5. **응급 알림** — 위험 패턴 감지 시 보호자 알림

---

## 6. 차별점

- **갤럭시/애플 헬스 대비:** 사후 기록이 아닌 **사전 예측**
- **병원 PSG 검사 대비:** 일회성이 아닌 **24/7 일상 모니터링**
- **운전자용 졸음감지 대비:** 단발성이 아닌 **개인 baseline 학습 기반 패턴 분석**

## 9. 차별화 무기 (Phase 2, 최소 3개 목표)

대기업/연구직이 좋아하는 키워드. 시간 봐가며 우선순위 정해 적용.

1. **개인화 baseline 학습** — 사용자별 정상 범위 보정 (이게 "개인화 헬스케어 AI"의 핵심)
2. **Self-supervised pre-training** — 라벨 없는 데이터로 사전학습 (대기업 최애 키워드)
3. **실시간 스트리밍 처리** — Kafka 또는 Redis Streams
4. **모델 경량화** — ONNX/TFLite, 온디바이스 추론 (삼성 헬스 직격)
5. **MLOps 파이프라인** — MLflow + 자동 재학습

## 14. 서류·면접 전략 (핵심 조언)

### 개인 프로젝트, 서류에 100% 쓸 수 있다
- **프로젝트 경험 칸** + **자기소개서(전문성 항목 1500자)** + **포트폴리오 링크**
- 경력란이 빈 건 신입에게 정상. "프로젝트 경험" 섹션이 따로 있음.
- 개인 프로젝트가 학교 캡스톤보다 강하게 평가되기도 함 (본인 주도성·지속성)

### 회사가 신입에게 보는 5가지
1. 일 시킬 수 있나 (기본기·완주경험·문제해결)
2. 도메인 알고 오나
3. 같이 일할 만한가 (소통·근거)
4. 성장 가능성
5. 가짜 아닌가 (본인이 진짜 했나)

### "오오" 만드는 5조건
1. 문제 정의 구체적 + 본인 언어로 설명 ✅ (Hena 보유)
2. 기술 결정에 근거 (→ ADR 기록 필수)
3. 베이스라인부터 점진 개선 (XGBoost→CNN→LSTM→SSL)
4. end-to-end 완성도 (데이터~서빙~Docker~테스트)
5. 문서화 + 데모 (README·다이어그램·영상)

### 신뢰도 빌딩 (개인 프로젝트의 약점 보완)
1. GitHub 꾸준한 커밋 (잔디 = 증거)
2. 커밋 메시지 잘 쓰기 (feat:/fix: 컨벤션)
3. 블로그/Notion 진행 기록 (격주라도)
4. README 정성껏
5. 가능하면 공모전/학회 포스터 1개 (외부 인증)

### 보험 전략 (Hena에 다 걸지 않기)
- 메인: Hena 1.5년
- 보험1: 코딩테스트 (4학년 1학기~, 별개)
- 보험2: CS 기초 (OS·네트워크·DB, 학교 수업)
- 보험3: 학부연구생 (Hena와 시너지)
