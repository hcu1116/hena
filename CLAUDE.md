## 프로젝트 한 줄 소개

**HENA** — 기면증을 비롯한 희귀 수면·신경 질환의 정보를 제공하고,
웨어러블 기술로 환자의 일상 위험(졸음·탈력발작)을 미리 감지하는 가능성을 탐구하는 프로젝트.

- 개발자: 황찬웅 (가천대학교 인공지능학과 3학년)
- 시작일: 2026-05-27
- 동기: 기면증 환자가 일상에서 겪는 위험을 기술로 줄일 수 있다는 가능성에서 출발
- 포지셔닝: 의료 "진단"이 아닌 "스크리닝 + 전문의 상담 권유"

## 홈페이지 설계 원칙

### 포지셔닝 / 톤
- 프리미엄 다크 헬스테크. 어둡고 절제됐지만 고급스럽고 감각적인 분위기.
- 레퍼런스 톤: Levels, WHOOP, Oura 같은 사이트의 "느낌"만 참고 (절대 그대로 베끼지 말 것 — HENA만의 디자인으로).
- 핵심 두 가지: "딱 봤을 때 고급스럽다" + "스크롤할수록 재밌다".

### 색 (절제가 생명)
- 베이스: 블랙 + 그레이 톤.
- 강조색: Electric Blue 딱 하나. 진짜 중요한 곳에만 콕 찍기.
- 여러 색 칠하지 말 것 = 산만 = 아마추어 느낌.

### 고급스러움을 만드는 6가지 규칙 (엄격히 준수)
1. 여백을 넉넉하게. 꽉 채우지 말 것 (꽉 차면 전단지처럼 싸 보임).
2. 타이포 위계를 크게. 헤드라인은 과감하게 크고, 본문은 작고 차분하게. 이 크기 대비가 멋을 만든다.
3. 색은 끝까지 절제 (위 참고).
4. 모션은 느리고 부드럽게. 스크롤 시 요소가 천천히 떠오르는(fade / slide-up) 정도. 빠르고 통통 튀는 모션 금지 (싸구려 느낌).
5. 이모지·클립아트 아이콘 금지. 꼭 필요하면 얇은 라인 아이콘만, 아니면 타이포로 해결.
6. 가운데 정렬만 반복하지 말 것. 좌우 비대칭 배치로 리듬을 줄 것.

### 구조 = 스크롤 스토리보드 (섹션마다 레이아웃이 전부 달라야 함)
1. 히어로 — 풀스크린. 압도적 첫 화면 + 한 줄 후킹 카피.
2. 문제 제기 — 센터 타이포. 보이지 않는 병, 진단까지 수년.
3. 작동 원리 — 분할 + 다이어그램. 오렉신 결핍 → 자율신경 이상 → HRV → 웨어러블로 측정 가능.
4. HENA 사용 흐름 — 가로 스텝. 착용 → 측정 → 스크리닝 → 전문의 상담 권유.
5. 라이브 시뮬레이션 (이 페이지의 주인공) — 인터랙션. 일반인 vs 기면증 HRV를 눈으로 비교. 느리고 차이가 명확하게.
6. 근거 / 데이터 — 분할. 실제 PhysioNet 데이터 + ML 결과 + 솔직한 한계(웨어러블 HRV ≠ 임상 HRV).
7. 질환 알아보기 — 카드 그리드. 질환 정보 포털 입구.
8. 면책 + 행동 유도 — 센터 CTA. "진단 아님, 전문의 상담 권유".

### 불변 규칙
- 모든 의학 정보에 "진단 아님, 전문의 상담 권유" 면책 문구 유지.
- 기존 기능(실시간 심박/HRV 시뮬, 질환 정보, ML 데이터)은 버리지 말고 위 구조에 맞게 재배치.
- 영상은 지금은 고급 그라데이션/추상 애니메이션 배경으로. 나중에 무료 스톡 영상으로 교체 가능하게 설계.
